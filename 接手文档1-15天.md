## 七、关键设计决策(必须遵守,不要质疑/重构)

原始 10 条(Day1–5):
1. 三层 API 鉴权:`X-App-Key + X-Timestamp(5分钟容差) + X-Sign(HMAC-SHA256)`
2. 配额扣减:Redis Lua 脚本原子 `GET→比较→INCR`,Key TTL=2 天
3. app_secret 只创建时返回一次,GET 接口手动 setAppSecret(null)
4. 构造方法注入(@RequiredArgsConstructor)替代 @Autowired 字段注入
   (例外:Day12 AbstractMockChannelHandler 抽象基类共享依赖用 @Autowired 字段注入,见决策19)
5. 路径分离:`/api/v1/**` 后台(无鉴权) vs `/api/external/**` 业务方走拦截器
6. 模板变量正则 `\{\{(\w+)}}` 自动提取,渲染缺变量抛 BusinessException(PARAM_INVALID 40001)
7. pulse-common 依赖严格设 provided,避免污染下游
8. GlobalExceptionHandler:BusinessException + MethodArgumentNotValidException + 兜底 Exception
9. Redisson 配置在根级 `redisson:`,不嵌套 spring 下
10. MyBatis-Plus 用 lambdaQuery(),避免硬编码字段名

Day6–10 决策:
11. **发送异步化**:RocketMQ 解耦,controller 投 MQ 立即返回 messageIds,消费端处理。
    topic=pulse_message_send,producer group=pulse_producer_group,consumer group=pulse_message_consumer_group
12. **一个接收者 = 一条 MQ 消息**(一个 MessageTask):每人独立频控/路由/轨迹/重试,不把 List 塞一条
13. **模板渲染留在 controller(同步)**:variables 全接收者共用渲染一次;模板不存在/缺变量同步快速失败;
    MessageTask 携带已渲染好的 content,不带 variables
14. **messageId = "MSG_" + 雪花**,controller 入队时生成,作为幂等/轨迹/死信主键
15. **敏感词 DFA Trie**:@PostConstruct build 进内存,检测不查库;volatile 整树原子替换热更新;
    大小写不敏感;O(文本长度)与词库大小无关
16. **频控 ZSet 滑动窗口**:一个 ZSet(score=毫秒时间戳)同时支撑 1分钟/1小时/1天三窗口;
    Lua 原子;key=pulse:freq:{appId}:{receiver};member=now+UUID 保唯一
17. **消息状态机 + 双表**:t_message=当前快照(status被覆盖),t_message_track=历史流水(只追加);
    updateStatus 用 CAS 条件更新(WHERE status=from)天然防重复消费;返回 boolean(是否推进成功)

Day11–15 新增决策:
18. **渠道成功率双 key**:pulse:channel:total:{type}:{provider} 全量 + pulse:channel:fail:{type}:{provider} 仅失败,
    失败率=ZCARD(fail)/ZCARD(total) 都是 O(1)。不用单ZSet存S_/F_前缀(那样数失败要拉全量循环判前缀 O(N),
    高QPS热路径拖垮Redis)。member=now+UUID,5分钟滑窗,TTL600,record/query 各一段 Lua
19. **路由决策三要点**:①失败率>20%自动跳过该provider ②样本下限MIN_SAMPLES=10(不足当健康,
    避免1/1=100%误判)③全部超阈值时降级到"最不坏"的而非丢弃消息;候选为空才抛CHANNEL_UNAVAILABLE(60301)。
    抽象基类共享依赖破例用@Autowired字段注入(基类构造注入会牵连所有子类写super,折中)
20. **策略模式+工厂+模板方法多渠道**:ChannelHandler接口(support+send),四渠道实现;
    ChannelFactory 注入List<ChannelHandler>自动收集+EnumMap按type索引,加渠道=加@Component类不改工厂(开闭);
    AbstractMockChannelHandler 基类管故障注入+随机判定(模板方法),子类填doMockSend
21. **邮件真发脱离基类**:EmailChannelHandler 不extends基类而implements接口自己send,
    成败由JavaMail发送结果决定(不能随机判定);保留故障注入开关供降级测试;账密放yml(spring.mail.*),
    from=@Value(username)。短信/推送/微信保持Mock(短信签名需企业资质,生产接阿里云)
22. **重试指数退避+死信**:失败→延时重试(RocketMQ delayLevel 1s/5s/30s/2m/10m=level 1/2/4/6/14),
    5次耗尽→t_dead_letter+状态RETURNED(4);retryCount靠MQ消息体传递+库同步;
    重试时CAS前置态从FAILED取(不是PENDING);消费端开头advanced接updateStatus返回值防重复消费

## 八、当前发送链路

### Controller(ExternalSendController.send,/api/external/v1/send,同步快速失败)
```
鉴权(拦截器:Key+Timestamp+Sign,塞 appId)
  ↓
1. 扣配额 quotaService.deductQuota(appId)         // 每请求扣1(TODO Day16:按接收者扣+防超卖)
2. 查模板 getByCodeCached(appId, templateCode)    // 不存在抛 TEMPLATE_NOT_FOUND
3. 渲染 templateRenderer.render(content, variables) // 缺变量抛 PARAM_INVALID(40001)
3.5 敏感词 auditService.check(renderedContent)     // content级,循环外,命中抛 60201
  ↓ for receiver in receivers:
4.1 频控 freqLimitService.checkAndRecord(appId, receiver)  // receiver级,循环内,超频抛 60203
4.2 messageId = "MSG_" + snowflake.nextId()
4.3 落库 saveMessage(...)                          // status=PENDING(0),写初始轨迹,retry_count=0
4.4 投 MQ messageProducer.sendTask(task)           // task.retryCount 默认0
  ↓
5. 立即返回 {status:accepted, count, messageIds}
```

### Consumer(MessageConsumer.onMessage,Day15 重写)
```
retryCount = task.retryCount(默认0)
按 retryCount 选前置态:首次PENDING / 重试FAILED → 推进 SENDING(advanced 防重复消费,false则return)
  ↓ try:
查模板拿 channelType → 路由 route(选provider) → 策略 factory.get(type).send(ctx) → 埋点 record
  成功 → SENDING→SUCCESS,return
  失败 → lastError 记下,进 handleFailure
  ↓ handleFailure:
SENDING→FAILED
retryCount<5 ? (retryCount+1, 库同步, producer.sendRetryTask 延时投递) : moveToDeadLetter(写死信+状态RETURNED)
```

## 十一、Redis Key 命名规范
- 配额:`pulse:quota:{appId}:{yyyyMMdd}`,TTL 2天
- 应用缓存:AppService 按 key/id 缓存(getByAppKeyCached / getByIdCached)
- 模板缓存:TemplateService getByCodeCached,10分钟 TTL
