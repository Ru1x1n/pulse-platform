# Pulse - 企业级消息推送中台

> 一个统一对接多通道(短信/邮件/Push/微信)的 SaaS 消息推送平台,
> 业务方一行 API 调用搞定多渠道发送,告别每个渠道 SDK 接入的痛苦。

## 项目目的

模拟阿里云 / 极光推送 / 个推等商业消息推送服务的核心架构,
解决企业级消息推送中的四个核心问题:
**多渠道接入复杂、渠道故障切换慢、配额计费难管理、消息追踪不可观测**。

## 核心特性

- 🔐 **三层鉴权**:API Key + HMAC 签名 + 时间戳防重放
- ⚡ **智能降级**:基于 Redis 滑动窗口的实时成功率,自动跳过失败渠道
- 🔁 **指数退避重试**:RocketMQ 延时消息实现 1s/5s/30s/2m/10m 重试链
- 📊 **全链路追踪**:Canal 监听 binlog → ES 实时索引,支持多维查询
- 🛡️ **多级限流**:Sentinel 接口限流 + Redis 用户频控 + DFA 敏感词
- 🎯 **AB 测试**:模板版本化 + 流量分流 + 漏斗转化统计
